what is Docker?

Docker is a tool that lets you package an application + all its dependencies (libraries, code, runtime) into one unit called a container.


what are Containers?

A container is a lightweight, isolated environment where your app runs.

It includes:

Code
Libraries
Dependencies

Example:
You build a Node.js app → put it in a container → it will run the same on any system.

But it shares OS kernel, so it’s fast.



Difference between VM and container?
VM (Virtual Machine) = full OS + app + dependencies
Container = app + dependencies (shares OS kernel)

note: Containers share the OS kernel, and that’s what makes them fast.


Benefits of Docker:
1. Portability: Run your app anywhere (dev, staging, production) without worrying about environment differences.
2. Consistency: Same environment across all stages of development and deployment.
3. Isolation: Each container runs independently, so you can run multiple apps on the same host without conflicts.
4. Scalability: Easily scale your app by running multiple containers.


Docker Images = Blueprint for creating containers (read-only template)
Docker Containers = Running instances of Docker images (read-write)

Dockerfile = Text file with instructions to build a Docker image

example:
Docker Image = Recipe 📖
Docker Container = Dish 🍜
Dockerfile = Instructions to create recipe 🧾

Dockerfile → Docker Image → Docker Container


commands:

basics
1. docker --version → check Docker version
2. docker images → list all Docker images
3. docker ps → list running containers
4. docker ps -a → list all containers (running + stopped)


5. docker build -t myapp . -> build Docker image from Dockerfile in current directory and tag it as "myapp"
6. docker run -p 3000:3000 myapp -> run container from "myapp" image and map port 3000 of container to port 3000 on host machine
7. docker stop <id> -> stop running container
8. docker rm <id> -> remove stopped container
9. docker rmi <id> -> remove Docker image
10. docker create <image_name> -> to create a container

in class notes 
docker pull nginx:latest -> pull latest nginx image from Docker Hub
docker create --name nginx-container nginx:latest -> create a container named "nginx-container" from the nginx image
docker pause nginx-container -> pause the container
docker unpause nginx-container -> unpause the container
docker rm nginx-container -> remove the container
docker run nginx:latest -> run a container from the nginx image
docker run -d --name nginx-container nginx:latest -> run a container in detached mode, name it "nginx-container", and map port 80 of container to port 80 on host machine
docker exec -it nginx-container /bin/bash -> access the container's shell
