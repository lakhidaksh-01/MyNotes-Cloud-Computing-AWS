IAM - Identity and Access Management
IAM is a service that controls authentication and authorization in AWS. It allows us to securely manage users, groups, roles, and permissions using policies, while following the principle of least privilege.


Root user = full access to all AWS services and resources (not recommended for daily use)


IAM Users = individual users with specific permissions, individual users (like you, devs, admins)
Each has: Username + password (console)
          Access keys (CLI/API)

Example: Daksh = IAM user → only allowed EC2, not RDS



IAM Groups = Collection of users
Permissions assigned to group (not individual)

Example:
“Developers” group → EC2 access
Add 10 users → all get same access

Note : Single user can be in multiple groups, permissions are cumulative.



IAM Roles = IAM Role is a temporary identity with specific permissions that can be assumed by users or AWS services to access resources without using credentials.

IAM Roles can be assumed by AWS services, IAM users, or other AWS accounts to get temporary access to resources.

Example:
1. EC2 wants to access S3
2. Normal user → assumes Admin role for limited time
3. Company A accesses Company B resources via role



IAM Policies = JSON document that defines permissions



POLP - Principle of Least Privilege: Grant only the permissions necessary for users to perform their tasks.

Notes: Policies define what actions are allowed or denied, while the Principle of Least Privilege ensures that only the minimum required permissions are granted when creating those policies.
